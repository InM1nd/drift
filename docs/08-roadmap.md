# 08 · Роадмап

Оценки — для компактного прототипа (1 класс, 1 акт, ~30 карт), в днях активной работы солопроекта. Реальный календарный срок обычно растягивается на 6–11 недель в режиме хобби.

| Фаза | Что делаем | Дни |
|---|---|---|
| 0 · Настройка | Vite + React + TS проект, git, базовые типы `CardData`/enemy/relic/potion | 1–2 |
| 1 · Боевое ядро | Рука, Заряд, intent врага, XState-автомат боя, сидированный RNG, dev-панель, 8–10 тестовых Протоколов | 5–7 |
| 2 · Заход | Граф узлов остова, переход между узлами, экран награды, снапшот-сохранение | 4–5 |
| 3 · Контент | Все 30 Протоколов (+ прокачанные версии), 4 Модуля, 6 Инъекторов, Терминал снабжения, Ремонтный отсек | 6–9 |
| 4 · Враги | 3 рядовых, 1 Страж (элита), Ядро-Страж с фазами — паттерны из [04 · Враги](04-enemies.md), первый баланс-пасс | 4–5 |
| 5 · Арт и сочность | Пиксель-спрайты вместо плейсхолдеров (см. [01 · Сеттинг](01-setting.md) — визуальный ориентир), Framer Motion, звук, тряска экрана | 4–6 |
| 6 · Мобильный UI и PWA | Портретная раскладка, touch-таргеты, манифест, service worker, тест на реальном телефоне | 3–4 |
| 7 · Плейтест | Баланс, баги, [Уровень угрозы](11-threat-level.md) | 3–5 |
| 8 · Деплой | Хостинг, установка на домашний экран (iOS/Android/desktop) | 1–2 |

## Инструменты

| Инструмент | Роль |
|---|---|
| Vite + React + TS | Сборка и фреймворк |
| vite-plugin-pwa | Манифест, service worker, офлайн-кеш «из коробки» |
| XState + Zustand | Машина состояний боя и хранилище состояния захода |
| Framer Motion | Твины карт, hover, layout-анимации |
| Aseprite | Пиксель-арт и спрайт-анимации |
| jsfxr / freesound.org | Ретро-звуки и SFX |
| Git + Vercel | Версионирование; статический деплой SPA, знакомая платформа |

## Минимальный числовой бейзлайн

Числа, которых не хватало для старта Фазы 1 — не финальный баланс, а отправная точка, которую переиграет dev-панель и плейтест:

| Параметр | Значение |
|---|---|
| HP Ныряльщика | 70 |
| Стартовая колода | 5× Залп + 4× Барьер (9 карт) |
| Размер руки | 5 карт за ход |
| Стартовые кредиты | 99 |
| Награда за Отсек | 10–20 кредитов + выбор Протокола |
| Награда за Стража (элита) | 25–35 кредитов + Модуль |
| Цены в Терминале снабжения | Протокол ~50–75 · Модуль ~150–200 · Инъектор ~20–35 · удаление карты ~75–100 (растёт с каждым разом) |

HP/урон врагов и босса уже зафиксированы в [04 · Враги](04-enemies.md) — их пересчитывать не нужно.

## Текущий статус (сверено с кодом и git log, не только с намерением)

1. ✅ **Фаза 0 — скаффолд.** Vite + React + TS, git, зависимости, базовые типы. `src/types/`.
2. ✅ **Числовой бейзлайн выше** — зафиксирован.
3. ✅ **Фаза 1 + Milestone A/B — боевое ядро и заход.** Розыгрыш карт, урон/блок/статусы, конец хода, victory/defeat, dev-панель, граф узлов забега, награды, снапшот-сохранение. Плейтест урезанного среза пройден — решение вкладываться в полный контент принято и уже реализовано (см. пункт 4).
4. ✅ **Фаза 3 — контент.** `src/data/cards.ts` содержит 60 записей (30 Протоколов + «+»-версии), `modules.ts` — 4 Модуля, `injectors.ts` — 6 Инъекторов — числа совпадают с таргетом ниже. Терминал снабжения и Ремонтный отсек реализованы.
5. ✅ **Фаза 4 — враги.** 3 рядовых + Страж (элита) + Ядро-Страж с фазами и summon-подкреплением, аудит ростера против [04 · Враги](04-enemies.md) пройден (коммит `0e0cf0a`).
6. ✅ **Фаза 6 — PWA.** Манифест, service worker (кеш только app shell), install-иконки, `vercel.json` (коммит `f8c8cd0`). Портретная раскладка есть; отдельный проход по touch-таргетам на реальном телефоне не подтверждён — открытый пункт.
7. ✅ **Фаза 7 — Уровень угрозы.** Движок/сторы и UI-селектор подключены (см. [11 · Уровень угрозы](11-threat-level.md) — этот файл раньше врал, что UI не подключён, синхронизировано).
8. 🔶 **Фаза 5 — арт и сочность, в работе.** HUD-визор (визуальный стиль A) активно дорабатывается («Phase 9: HUD v3», коммит `e10f11c`) — общий system-notice слой, отзывчивость экранов. Оба стиля из [09](09-style-hud.md)/[10](10-style-pixel.md) остаются переключаемыми кандидатами (`data-visual-style`), финальный выбор одного стиля ещё не зафиксирован.
9. ⬜ **Фаза 7 (доки) — полноценный плейтест и баланс-пасс** — не подтверждён.
10. ⬜ **Фаза 8 — деплой** — `vercel.json` и сборка готовы (проверено `npm run build`/`preview`, Фаза 6), но фактический деплой/финальная проверка на устройствах не подтверждены.

Отдельно, не блокирует последовательность выше:
- Название и иконки для PWA-манифеста — решены (Фаза 6).
- ✅ **Стартовый экран и игровое меню.** `src/components/StartScreen.tsx` — титульный экран (показывается всегда, включая холодный старт без сейва — авто-старт убран), `src/components/GameMenu.tsx` — пауза/меню, доступное из любого экрана забега. Визуальный стиль (`hud`/`pixel`) переехал из dev-only переключателя в шапке в player-facing `src/components/SettingsPanel.tsx` (хранится в `src/state/settingsStore.ts`), туда же переехали dev-инструменты (см. «Dev-инструменты» в [07 · Архитектура](07-architecture.md)).
- Полноценный обучающий онбординг (туториал по механикам, а не просто титульный экран) — по-прежнему сознательно отложен: для прототипа тестировщик ты сам, задача релизной стадии.
